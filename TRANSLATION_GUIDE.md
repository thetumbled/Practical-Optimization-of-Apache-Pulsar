# GitBook 章节翻译指南

本文档提供了将中文章节翻译成英文的完整流程。

## 翻译流程概览

将 GitBook 章节从中文翻译到英文需要处理三个关键部分：

### 1. **翻译内容文件**
   - 读取中文源文件（所有 .md 文件）
   - 忠实翻译原文，保持技术术语准确性
   - 保留代码块、链接、表格等格式
   - 保留 GitBook 特殊语法（如 `{% embed url="..." %}`）

### 2. **处理图片资源**
   - 复制中文图片到英文 assets 目录
   - 使用唯一命名避免冲突（如 `image (chX-Y).png` 格式）
   - 更新所有 markdown 文件中的图片引用路径

### 3. **更新目录结构**
   - 修改 `doc/english/SUMMARY.md` 文件
   - 添加新章节的完整目录结构


## 故障排查

### 问题：GitBook 不显示新章节
**解决方案**：检查 `doc/english/SUMMARY.md` 是否正确添加了章节目录

### 问题：图片显示不出来
**解决方案**：
1. 检查图片是否已复制到 `doc/english/.gitbook/assets/`
2. 检查 markdown 文件中的图片路径是否正确更新
3. 确认图片文件名与引用路径完全匹配（包括括号、空格）

### 问题：图片命名冲突
**解决方案**：使用章节特定的前缀（如 ch7-, ch8-）避免冲突
